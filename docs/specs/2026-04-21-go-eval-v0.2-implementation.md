# go-eval v0.2 Implementation Spec

## Summary

v0.2 ships the smallest high-value slice:
- OpenAI adapter promotion into `adapters/openai`
- deterministic metrics (`Contains`, `Regex`, `JSONPath`, `FieldCount`)
- strict `Compound` metric (single JSON shape, one judge call, one retry)
- `Precheck` metric wrapper for short-circuiting expensive judge calls
- explicit result sinks in `Runner` via options

Deferred to v0.3:
- conversation-case APIs
- YAML loader submodule
- compare/regression package
- non-OpenAI adapters

## Contract Changes

1. `Judge` remains unchanged.
2. Optional raw-judge extension added:
   - `type RawJudge interface { EvaluateRaw(context.Context, string) (RawJudgeResponse, error) }`
   - `type RawJudgeResponse struct { Content string; Tokens int }`
3. `Result` adds:
   - `Dimensions []DimensionResult` for structured compound output
   - unexported sentinel field to discourage external unkeyed literals
4. `Runner` adds sink option:
   - `WithResultSink(ResultSink)`
   - `DefaultResultSink()` helper reads `GOEVAL_RESULTS_DIR`

## Feature Details

### OpenAI adapter

- New module: `adapters/openai`
- Must satisfy both `eval.Judge` and `eval.RawJudge`
- 30s timeout, temperature `0`
- `Evaluate` parses score JSON, retries once with `JSON only, no prose`

### Deterministic metrics

- `Contains`: substring match (`Output` contains `Expected`)
- `Regex`: safe compile (`regexp.Compile`), invalid pattern returns failed result (non-error)
- `JSONPath`: constructor-only validation (`NewJSONPath`, `MustJSONPath`)
  - supported grammar: dot-separated keys + `[N]` non-negative indexes
  - unsupported syntax fails fast
- `FieldCount`: min non-null top-level JSON fields; score capped at `1.0`

### Compound metric

- New `Dimension` + `Compound`
- Judge response shape accepted: nested JSON only
  - `{"dimension": {"score": 0.0..1.0, "reason": "..."}}`
- Flat format and out-of-range scores are rejected
- One retry max with strict JSON suffix
- Result:
  - `Score`: mean of dimension scores
  - `Passed`: all non-zero thresholds pass
  - `Dimensions`: structured per-dimension output
  - `Reason`: concise machine-friendly summary string

### Precheck wrapper

- New composable wrapper metric:
  - `type Precheck struct { Pre Metric; Main Metric }`
- If `Pre` fails, `Main` is skipped and zero-score failed result is returned
- Reason prefix: `precheck<PreName> failed: ...`

### Result sink

- `Runner` writes `RunResult` only when a sink is configured
- Sink write errors call `tb.Errorf` and do not block metric completion
- Default sink writes JSONL and is concurrency-safe for `t.Parallel`

## Test Requirements

1. Deterministic metric unit tests:
   - positive and negative cases
   - invalid regex and JSON parse/path failures return failed result, non-error
2. Compound tests:
   - all pass, one fails, threshold-zero skip
   - markdown-fenced response
   - reject flat response, out-of-range score, missing dimensions
   - verify one-retry behavior
3. Precheck tests:
   - skip main metric when precheck fails
   - confirm reason prefix
4. Sink tests:
   - writes when configured
   - no writes on metric fatal errors
   - concurrent writes are race-safe
5. Adapter tests:
   - parser unit tests
   - integration test behind `//go:build integration`, skip without `OPENAI_API_KEY`

## Compatibility and Constraints

- Core package remains stdlib-only.
- Existing `GOEVAL` skip gating remains unchanged.
- Existing v0.1 metrics keep behavior and tests.
